# gstack 사용법

## 개요

`gstack`은 저장소를 내려받은 뒤 `setup`을 한 번 실행해야 실제로 스킬이 등록됩니다.

중요한 점:

- Windows에서는 `setup`이 bash 스크립트이므로 `PowerShell`보다 `Git Bash` 또는 `WSL`에서 실행하는 것이 안전합니다.
- Windows에서는 `Bun`뿐 아니라 `Node.js`도 필요합니다.

## 준비물

다음이 설치되어 있어야 합니다.

- `Git`
- `Bun`
- `Node.js`

## 1. Codex에서 사용하는 방법

현재 이 저장소를 이미 내려받은 상태라면 다시 클론할 필요는 없습니다.

`Git Bash`를 열고 아래 명령을 실행합니다.

```bash
cd /d/work_ai2/GarryTan/gstack
./setup --host codex
```

이 명령은 다음 작업을 수행합니다.

- 필요한 바이너리 빌드
- Playwright Chromium 설치
- Codex용 스킬을 `~/.codex/skills/gstack-*` 아래에 등록

설치가 끝난 뒤에는 Codex에서 gstack 스킬을 이렇게 사용할 수 있습니다.

- `Run the gstack-review skill on this repository`
- `Run the gstack-office-hours skill`
- `Run the gstack-qa skill`

## 2. Claude Code에서 사용하는 방법

Claude Code는 보통 `~/.claude/skills/gstack` 위치에 설치해서 사용합니다.

```bash
git clone --single-branch --depth 1 https://github.com/garrytan/gstack.git ~/.claude/skills/gstack
cd ~/.claude/skills/gstack
./setup --no-prefix
```

설치가 끝나면 Claude Code에서 다음처럼 실행합니다.

- `/office-hours`
- `/review`
- `/qa https://your-staging-url`
- `/ship`

## 3. 프로젝트 안에 로컬 설치하는 방법

팀 프로젝트에 같이 넣고 싶다면 프로젝트 안의 `.claude/skills/`로 설치할 수 있습니다.

```bash
cd /path/to/your-project
/path/to/gstack/setup --local --no-prefix
```

## 4. 가장 짧은 요약

- Codex에서 사용할 경우: 현재 저장소에서 `./setup --host codex`
- Claude Code에서 사용할 경우: `~/.claude/skills/gstack`에 설치 후 `./setup --no-prefix`
